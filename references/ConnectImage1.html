Flare Network

Pangolin on the Flare Network
Flare is a blockchain which presents developers with a simple and coherent stack for decentralized interoperability.

Introduction

Flare is a layer 1 EVM blockchain that has 2 core protocols, the State Connector and Flare Time Series Oracle (FTSO). These protocols allow developers to create an ecosystem of robust and decentralized interoperability applications.
That’s why we say #ConnectEverything.

Core Protocols

Layer-1 with native data and state acquisition.

​
✅
 The fundamental protocols of the Flare blockchain, available to developers at launch.
FTSO
State Connector
Reliable decentralized prices
The FTSO utilizes the network structure to deliver highly decentralized prices and data series to dapps on Flare, without relying on centralized providers.
​Learn more​
Secure state acquisition from other blockchains
The State Connector allows information from other chains to be securely and trustlessly used on Flare.
​Learn more​

Scalable EVM-based smart contracts

Flare’s scalable EVM allows existing EVM tooling and Solidity to be used to launch any existing or envisaged EVM dapp on Flare. Learn more​

Fast, low fee and low carbon

Flare’s next generation technology brings fast, secure and efficient transactions with very low gas fees and a small carbon footprint.

In development

​
☑
 Additional functionality being developed by ecosystem partners.
FAssets
Layer Cake
Non-smart contract token bridging
FAssets will enable non-smart contract tokens like BTC, XRP, DOGE and others to participate in DeFi on Flare and bridge to other chains.
​Learn more​
Insured bridging
Layer Cake will be a material advance in the security, speed and scalability of bridging between smart contract networks, addressing the core vulnerabilities of existing approaches.
​Learn more​

Cross-chain messaging by relay

Secured and staked relay of data between multiple chains, including Flare, will provide secure cross chain composability and interoperability.

Flare Rollout Process

Flare genesis was 14th July 2022. The network is now in Observation Mode while validators are onboarded. Once this process is complete, the Token Distribution Event can occur.
